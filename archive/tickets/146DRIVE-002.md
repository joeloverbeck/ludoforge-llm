# 146DRIVE-002: Implement drive function + four-shadow chain in `kernel/microturn/drive.ts`

**Status**: COMPLETED
**Priority**: HIGH
**Effort**: Large
**Engine Changes**: Yes — new file `packages/engine/src/kernel/microturn/drive.ts`
**Deps**: `archive/tickets/146DRIVE-001.md`

## Problem

Spec 146 establishes that the FITL preview-pipeline's remaining ~17.8% performance gap (after the 64.76% gain from the fitl-preview-perf campaign) is dominated by per-iteration Zobrist `updateHash` calls in the synthetic-completion drive. The exp-017 attempt to thread a `skipFinalHash` boolean through 4 hot kernel functions was a wash — V8 deopt from parameter-shape changes offset the saved work.

This ticket implements the spec's solution: a dedicated kernel function `applyPreviewDriveGreedyChooseOne` that owns the drive loop end-to-end and uses a private mutable scoped state per Foundation 11's exception clause. It's accompanied by a four-function shadow chain that mirrors the canonical apply pipeline with `updateHash` calls bypassed, allowing the drive to canonicalize state only once at exit.

The new file is colocated to preserve the V8 JIT profiles of the canonical chain — the architectural argument from the spec's "Why a single new kernel function avoids the V8 problem" section.

## Architecture Check

1. **Cleaner than alternatives**: Per spec, three alternatives were rejected — (a) threading `skipFinalHash` through 4 hot functions (exp-017, V8 deopt offset gains), (b) lazy `state.stateHash` getter (~25 invasive call sites), (c) lowering `K_PREVIEW_DEPTH` (metric gaming). A separate file with shadow functions is the architecturally complete solution: it preserves the canonical chain's JIT profile, isolates the new optimization, and uses F#11's existing exception clause without inventing a new mutation primitive.
2. **Foundation 11 compliance**: The new function's working state is fully isolated to its synchronous scope. Caller-supplied `initialState` is never mutated (asserted in 146DRIVE-003). Intermediate unhashed states never escape the function's local scope. Final state is canonicalized via `updateHash` before return — external contract identical to the immutable path.
3. **Foundation 14 atomic-cut and mechanical uniformity**: This ticket is rated Large (~300 LoC across one new file) but the diff is mechanically uniform — each `*NoFinalHash` shadow is a line-by-line mirror of its canonical counterpart with `updateHash(def, X, runtime)` becoming `X` (the unhashed object). The Foundation 14 mechanical-uniformity exception applies.
4. **Agnostic boundaries preserved**: Drive function operates only on `GameDef`/`GameState`/`Decision` types — no game-specific identifiers, no FITL or Texas Hold'em branches. The function works for any GameDef whose chooseOne picker returns first-legal-supported (greedy semantics).
5. **No backwards-compat shim**: Pure addition. Canonical chain in `apply.ts` and `publish.ts` remains entirely unmodified — this is the architectural reason for the new file.

**Critical**: every branch end in this shadow must produce an unhashed object. Audit by grep — every `updateHash(` in the shadow source must be either inside the final canonicalization at the bottom of `applyPreviewDriveGreedyChooseOne` (1e below) or removed.

#### 1e. `applyPreviewDriveGreedyChooseOne` (the only `export const`)

Signature per spec D2 (parameters `def, initialState, origin, depthCap, runtime?`). Body:

1. Resolve runtime: `const resolvedRuntime = runtime ?? createGameDefRuntime(def);`
2. Initialize working state: `let workingState: GameState = initialState;`
3. Loop (mirror the cheap exit checks from `policy-preview.ts:732-752`):
   - Read `top = workingState.decisionStack?.at(-1)`. If undefined → break with `kind: 'completed'`, `depth: <current>`.
   - Extract `ctxKind = top.context.kind`, `topSeatId = top.context.seatId`.
   - If `ctxKind in {'actionSelection', 'outcomeGrantResolve', 'turnRetirement'}` OR `topSeatId !== origin.seatId` OR `top.turnId !== origin.turnId` → break with `kind: 'completed'`.
   - If `ctxKind === 'stochasticResolve'` → break with `kind: 'stochastic'`.
   - If `depth >= depthCap` → break with `kind: 'depthCap'`.
   - Greedy publish: `const greedy = publishMicroturnGreedyChooseOne(def, workingState, resolvedRuntime);`
   - If `greedy === null` → break with `kind: 'failed'`, `failureReason: 'noPreviewDecision'`.
   - Apply via shadow: `workingState = applyPublishedDecisionInternalNoFinalHash(def, workingState, greedy.microturn, greedy.decision, { advanceToDecisionPoint: true }, resolvedRuntime).state;`
   - `depth += 1;`
4. Final canonicalization: replicate the local `updateHash` helper logic from `apply.ts:59-72` to compute `workingState` with canonical `_runningHash`/`stateHash` via `computeFullHash`. Either inline the call or import `apply.ts`'s helper if it can be exported without breaking encapsulation. **Prefer inlining** to keep `apply.ts` unchanged — quote the same `computeFullHash`-based logic.
5. Return `{ state: workingState, depth, kind, failureReason? } as PreviewDriveResult`.

Wrap in `try`/`catch` matching the spec's failure semantics — caught errors return `kind: 'failed'` with truncated `failureReason`.
