# 147AOTCON-002: Extend compiled policy descriptors to full `AgentPolicyExpr` coverage and add determinism invariant

**Status**: COMPLETED
**Priority**: HIGH
**Effort**: Medium
**Engine Changes**: Yes тАФ kernel descriptor types, compiler lowering pass, agent runtime factory, schema artifacts, integration test
**Deps**: `archive/tickets/147AOTCON-001.md`

## Problem

147AOTCON-001 lands the `CompiledPolicyExpr` descriptor surface for the first supported expression families (the minimum subset defined in Spec 147 ┬зD1) plus the equivalence-scaffold test (D5.1). It deliberately populates `AgentPolicyCatalog.compiled.considerations` only. The full descriptor union must cover every `AgentPolicyExpr.kind` and every AST-bearing library expression carrier currently emitted by `lowerAgents` before the compiled path can be enabled as default in 147AOTCON-003. Coverage gaps include the four aggregate kinds (`zoneTokenAgg`, `globalTokenAgg`, `globalZoneAgg`, `adjacentTokenAgg`), the structural variants (`zoneProp`, `seatAgg`), the 22 operators inside `case 'op'`, and non-consideration carriers (`stateFeatures`, `candidateFeatures`, `candidateAggregates`, `pruningRules`, `tieBreakers`, and `strategicConditions`).

Spec 147 also requires a determinism invariant test (D5.2) that compiles the FITL and Texas catalogs twice and asserts byte-identical descriptor trees and identical `catalogFingerprint` values. This test is the F#8/F#16 proof that the lowering pass is referentially transparent.

## Assumption Reassessment (2026-04-26)

1. `packages/engine/src/agents/policy-evaluation-core.ts:446-562` confirms `evaluateExpr` switches over 10 top-level `expr.kind` cases plus 22 nested `op` operators (32 total dispatch arms). Every arm needs a parallel descriptor variant before 003 can delete the interpreter.
2. `packages/engine/src/kernel/types-core.ts:451` defines `AgentPolicyExpr` with the full union; the descriptor union from D1 must mirror it 1:1.
3. The existing `compiled-condition-equivalence.test.ts` pattern (with its `compiled-condition-production-helpers.ts` corpus builder) is the architectural template for both the equivalence test (introduced in 001) and the determinism test (introduced here).
4. `cnl/compile-agents.ts:117` computes `catalogFingerprint` via `fingerprintPolicyIr` against the assembled catalog object тАФ adding the new descriptor kinds to the lowering output before that line ensures fingerprint coverage without a new hash mechanism.
5. Post-001 review confirmed the live `lower-agent-considerations.ts` only populates `compiled.considerations`. 003's planned deletion of AST-bearing `library.*.expr` fields requires this ticket to add compiled descriptor equivalents for all other expression carriers before 003 starts.
6. Live reassessment on 2026-04-26 found FITL + Texas production policy catalogs do not currently contain every descriptor kind. Per Foundation #16, this ticket still owns automated semantic proof for production-absent runtime arms, but the proof must use generic synthetic expression fixtures rather than artificial production GameSpecDoc policy churn.

## Architecture Check

1. The descriptor union extension is mechanical: each new variant mirrors an existing `AgentPolicyExpr.kind`. No game-specific branches enter; the kernel materializes closures via the same fixed factory introduced in 001.
2. F#7 (Specs Are Data): every new descriptor variant and carrier map remains a JSON-serializable typed record. No `eval`, no `Function`, no closures stored on disk.
3. F#8 (Determinism): the determinism invariant test is the proof obligation for this ticket тАФ once added, future regressions in lowering-pass purity will fail CI.
4. F#13 (Artifact Identity): the extended descriptors flow through the same `fingerprintPolicyIr` pipeline as the rest of the catalog. No new hash mechanism is introduced.
5. F#14 (No Backwards Compatibility): this ticket extends but does not enable the compiled path. The AST fallback remains in place because 003 atomically removes it alongside the fixture migration. This intermediate state is part of the migration sequence, not a permanent shim.

## What to Change

### 1. Extend the `CompiledPolicyExpr` union in `kernel/types-core.ts`

Add the missing variants to the `CompiledPolicyExpr` union introduced in 001 (per Spec 147 ┬зD1):

- `{ kind: 'zoneProp'; zone: CompiledPolicyExpr | string; prop: string }`
- `{ kind: 'zoneTokenAgg'; zone: AgentPolicyZoneSource; owner: AgentPolicyZoneTokenAggOwner; prop: string; aggOp: AgentPolicyZoneTokenAggOp }`
- `{ kind: 'globalTokenAgg'; tokenFilter?: AgentPolicyTokenFilter; aggOp: AgentPolicyZoneTokenAggOp; prop?: string; zoneFilter?: AgentPolicyZoneFilter; zoneScope: AgentPolicyZoneScope }`
- `{ kind: 'globalZoneAgg'; source: AgentPolicyZoneAggSource; field: string; aggOp: AgentPolicyZoneTokenAggOp; zoneFilter?: AgentPolicyZoneFilter; zoneScope: AgentPolicyZoneScope }`
- `{ kind: 'adjacentTokenAgg'; anchorZone: AgentPolicyZoneSource; tokenFilter?: AgentPolicyTokenFilter; aggOp: AgentPolicyZoneTokenAggOp; prop?: string }`
- `{ kind: 'seatAgg'; over: 'opponents' | 'all' | readonly string[]; expr: CompiledPolicyExpr; aggOp: AgentPolicyZoneTokenAggOp }`

Keep the existing variants from 001 unchanged. The `op` variant from 001 already carries `op: AgentPolicyOperator` which covers all 22 operator names; no shape change is needed for `op` тАФ only the runtime factory (below) needs to handle each operator.

### 2. Extend the compiled catalog beyond considerations

Extend `CompiledPolicyCatalog` and its schema so the compiled side has a descriptor equivalent for every AST-bearing library carrier that 003 must later consume without `AgentPolicyExpr`:

- `stateFeatures[featureId].expr`
- `candidateFeatures[featureId].expr`
- `candidateAggregates[aggregateId].of` and optional `where`
- `pruningRules[ruleId].when`
- `considerations[considerationId].when` / `weight` / `value`
- `tieBreakers[tieBreakerId].value`
- `strategicConditions[conditionId].target` and optional `proximity.current`

Preserve each carrier's existing metadata (`type`, `costClass`, `dependencies`, aggregate `op`, consideration clamp/unknown behavior, etc.) so 003 can remove AST-bearing fields without inventing a second migration shape.

### 3. Extend `lowerAgentConsiderations` / compiled-policy lowering

Walk the missing `AgentPolicyExpr.kind` arms and every non-consideration carrier listed above. For each expression, emit the parallel `CompiledPolicyExpr` descriptor with the same field shape (no transformation тАФ these are 1:1 mirrors). Recurse into `seatAgg.expr` and `zoneProp.zone` (when the latter is itself an expression). Maintain referential transparency: same input AST тЖТ byte-identical descriptor.

### 4. Extend `buildPolicyExprClosure` in `agents/compiled-policy-runtime.ts`

Add one switch arm per new descriptor variant. Each arm captures the relevant `PolicyEvaluationContext` providers and dispatches to the existing evaluator helpers already used by `evaluateExpr`:

- `zoneProp` тЖТ `evaluateZoneProp(expr, candidate)` equivalent (read `policy-evaluation-core.ts:564+`)
- `zoneTokenAgg` тЖТ `evaluateZoneTokenAggregate(expr, candidate)` equivalent
- `globalTokenAgg` тЖТ `evaluateGlobalTokenAggregate(expr)` equivalent
- `globalZoneAgg` тЖТ `evaluateGlobalZoneAggregate(expr)` equivalent
- `adjacentTokenAgg` тЖТ `evaluateAdjacentTokenAggregate(expr, candidate)` equivalent
- `seatAgg` тЖТ `evaluateSeatAggregate(expr, candidate)` equivalent (recursive: build a closure for `expr.expr` and invoke per seat)

For `op` (already present from 001): extend the inner switch to cover all 22 operators (`add`, `sub`, `mul`, `div`, `min`, `max`, `abs`, `neg`, `eq`, `ne`, `lt`, `lte`, `gt`, `gte`, `and`, `or`, `not`, `if`, `in`, `coalesce`, `clamp`, `boolToNumber`). Each operator's closure body must reproduce the semantics from `evaluateExpr`'s `case 'op'` block тАФ same coercion rules, same `undefined` handling, same `RUNTIME_EVALUATION_ERROR` for division by zero.

Closures stay тЙд10 lines per arm to preserve V8 monomorphization (Spec 147 Risks).

### 5. Extend the equivalence corpus

Update `test/helpers/compiled-policy-production-helpers.ts` (introduced in 001) to surface considerations and non-consideration library carriers that exercise every production-reachable descriptor kind. The helper should walk the FITL and Texas catalogs and report a coverage summary by carrier and expression kind (total expressions ├Ч kinds covered) so the equivalence test can assert that every compiled carrier map is populated when the corresponding AST carrier exists. For descriptor kinds absent from current production catalogs, this ticket owns the smallest generic synthetic expression parity samples.

### 6. Regenerate schema artifacts

Run `pnpm -F @ludoforge/engine schema:artifacts` so `packages/engine/schemas/GameDef.schema.json` reflects the extended `CompiledPolicyExpr` union and every compiled carrier map under `AgentPolicyCatalog.compiled`.

### 7. Add determinism invariant test (D5.2)

Create `packages/engine/test/integration/agents/compiled-policy-determinism.test.ts`. For both FITL and Texas:

1. Compile the catalog twice (separate `lowerAgents` invocations).
2. Assert byte-identical `compiled` descriptor trees via canonical JSON serialization.
3. Assert identical `catalogFingerprint` values.

Marker: `// @test-class: architectural-invariant`. Failure means the lowering pass introduced nondeterminism (object key ordering, map iteration, etc.) тАФ the test is the proof for F#8.
