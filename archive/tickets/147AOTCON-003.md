# 147AOTCON-003: Enable compiled policy path as default, delete AST interpreter, regenerate fixtures, split hard-target miss

**Status**: COMPLETED — 2026-04-26; compiled-only migration landed, hard-target miss split to `tickets/147AOTCON-004.md`
**Priority**: HIGH
**Effort**: Large
**Engine Changes**: Yes — agent runtime, kernel types, schema, policy-catalog goldens, hard-target measurement
**Deps**: `archive/tickets/147AOTCON-002.md`

## Problem

After 147AOTCON-001 and 002, the `compiled` descriptor tree exists alongside the AST-bearing `library` shape and covers every `AgentPolicyExpr.kind`. The runtime still dispatches via the AST interpreter (`evaluateExpr` in `policy-evaluation-core.ts:446`) when the compiled tree is absent — a transitional state. Spec 147 §D4 and the Foundation 14 alignment in §D6 require the AST interpreter to be deleted, the optional marker on `AgentPolicyCatalog.compiled` to be removed, and repository-owned fixtures to be updated or validated **in the same change**, not as a sequence of compatibility steps.

This ticket also re-runs the hard-target measurement to prove whether Spec 147's performance hypothesis holds. The 2026-04-26 post-migration measurement did not reach the `25600 ms` hard target, and the CPU profile shows the remaining hotspot is dominated by GC and Zobrist hashing rather than the deleted policy AST interpreter. Per Foundations 15 and 16, the remaining hard-target work is split to `tickets/147AOTCON-004.md` instead of being hidden inside this ticket's Outcome.

## Assumption Reassessment (2026-04-26)

1. `policy-evaluation-core.ts:407-444` (`evaluateConsideration`) currently calls `evaluateExpr` three times per consideration (`when`, `weight`, `value`). After 002 lands, every consideration has a `CompiledPolicyConsideration` parallel — making `evaluateExpr` redundant once the compiled-only path is enabled.
2. `policy-evaluation-core.ts:781-823` (`resolveRef`) is only called from `evaluateExpr`'s `case 'ref'` arm (the agent confirmed no other callers in the reassessment). Deleting `evaluateExpr` orphans `resolveRef` — it must be deleted in the same change.
3. The policy-catalog goldens (`fitl-policy-catalog.golden.json`, `texas-policy-catalog.golden.json`) encode the AST-bearing `library` shape and must be regenerated to encode the compiled-only shape in the same change as the type narrowing. The policy-summary trace goldens do not serialize the agent catalog; they must be validated against the compiled-only runtime and only regenerated if their trace content changes.
4. The hard-target measurement command (`scripts/measure-preview-pipeline-hard-target.mjs`, added under archived ticket 146DRIVE-005) is the canonical methodology for Acceptance Criteria 1 and 5 — same script, same corpus, before/after profile comparison.
5. The library's `expr: AgentPolicyExpr` fields (on stateFeatures, candidateFeatures, aggregates, strategicConditions) are no longer read at runtime once the compiled path is the only path. The spec's Follow-On Tickets entry calls out "the AST-bearing `library` shape that's no longer needed" — this ticket removes those fields from the runtime types.
6. Hard-target proof correction: the compiled-only migration produced `mean_totalMs=27884.08` over three runs (`candidateBudget=465`, `sampledActionSelectionCount=50`) against `hardTargetMs=25600`, so the hard-target acceptance claim is false for the live repo. The top profile samples are GC and `fnv1a64`/Zobrist-family work, not the removed agent policy AST interpreter.

## Architecture Check

1. **Foundation 14 atomic cut**: The deletion of `evaluateExpr` + the agent-runtime `resolveRef` helper + AST-bearing library fields + policy-catalog fixture regeneration is a single coherent change. Splitting it would leave the build in a state where source and fixtures disagree (compiled-only types vs. AST-shaped fixture data). The policy-summary trace fixtures remain in the proof surface but are validated unchanged because they do not serialize the agent catalog.
2. **No transitional shim**: After this ticket lands, no `?` optional marker remains on `AgentPolicyCatalog.compiled`. `GameDef.agents`, when present, MUST carry a `compiled` field. Type-system-enforced — readers cannot accidentally fall through to a missing-compiled branch.
3. **Engine agnosticism**: The deletion path touches only kernel/agent types and game-agnostic compiled descriptors. No game-specific knowledge enters the engine.
4. **Determinism preserved**: The compiled path's referential-transparency proof (147AOTCON-002 D5.2) gates this ticket — if 002's determinism test fails, 003 must not land. Replay identity across 200-turn FITL traces is the runtime determinism gate.
5. **Foundation 15 / 16 split**: Because the hard-target miss is not in the AOT policy interpreter seam, this ticket must not claim the performance gate as satisfied. It may land the compiled-only cut only with the failed measurement recorded and `147AOTCON-004` owning the remaining root-cause work.

## What to Change

### 1. Make `compiled` required in `AgentPolicyCatalog`

In `packages/engine/src/kernel/types-core.ts`, change `AgentPolicyCatalog.compiled` from optional to required. Remove the `expr: AgentPolicyExpr` fields from `CompiledAgentStateFeature`, `CompiledAgentCandidateFeature`, `CompiledAgentAggregate`, and `CompiledStrategicCondition` (the library shapes that previously carried AST). The compiled descriptor tree is the only remaining expression carrier.

### 2. Delete the AST interpreter

In `packages/engine/src/agents/policy-evaluation-core.ts`:

- Delete `evaluateExpr` (lines 446–562).
- Delete `resolveRef` (lines 781–823) — orphaned by `evaluateExpr` deletion.
- Delete `evaluateExprList` and any other helpers exclusive to the AST path.
- Reduce `evaluateConsideration` (lines 407–444) to the compiled-only path: read `CompiledPolicyConsideration` from `this.input.def.agents.compiled.considerations[considerationId]`, materialize closures via `buildPolicyExprClosure` (147AOTCON-001's runtime), and evaluate. The dispatcher branch from D4 collapses to a single path.

Update the direct callers of `evaluateExpr` outside `policy-evaluation-core.ts` (per the reassessment inventory):

- `agents/policy-eval.ts` (2 tieBreaker call sites) — switch to `buildPolicyExprClosure`.
- `test/integration/agents/strategic-condition-e2e.test.ts` (11 test call sites) — switch to compiled-path equivalents.
- `test/unit/agents/policy-eval-strategic-condition.test.ts` (1 test call site) — same.

### 3. Update `lowerAgents` to drop AST emission

In `packages/engine/src/cnl/compile-agents.ts`:

- Stop emitting `expr: AgentPolicyExpr` fields in the library output (since the types no longer carry them).
- The `lowerAgentConsiderations` pass (introduced 001, extended 002) becomes the sole expression-emission path.
- `catalogFingerprint` still computed from the assembled catalog (now compiled-only) by `fingerprintPolicyIr`.

## Outcome

Completed 2026-04-26.

147AOTCON-003 landed the compiled-only agent policy runtime cut. `AgentPolicyCatalog.compiled` is now required, runtime policy evaluation reads compiled descriptors instead of falling back to the AST interpreter, and the private agent policy `evaluateExpr`/`evaluateExprList`/`resolveRef` path was removed. The compiler now emits stripped runtime `library` descriptors plus the required `compiled` tree, and the generated GameDef schema plus production policy-catalog fixtures were regenerated accordingly.

Post-review cleanup removed the remaining test-local AST-bearing fields from exported runtime library types (`when`, `weight`, `value`, `expr`, `of`, `where`, strategic-condition `target`/`proximity.current`) and moved those authoring-only shapes behind `packages/engine/test/helpers/policy-catalog-fixtures.ts`, so production kernel types no longer advertise stripped library AST fields.

The hard-target measurement was run and recorded, but did not pass: `previewOn_totalMs_ms=28946.57,27447.55,27258.11`, `mean_totalMs=27884.08`, `mad_pct=2.54`, `hardTargetMs=25600`, `pass=false`, `candidateBudget=465`, `sampledActionSelectionCount=50`. The post-migration CPU profile showed the remaining hotspot is GC and Zobrist/hash-family work (`fnv1a64`, `digestDecisionStackFrame`, `zobristKey`), not the deleted policy AST interpreter. The remaining performance work is explicitly owned by `tickets/147AOTCON-004.md`.

Trace policy-summary fixtures were validated unchanged through the engine test lane; only policy-catalog fixtures changed for the compiled-only shape. The live repo does not contain the originally drafted `scripts/sync-fixtures.sh`, so fixture regeneration used the built production fixture helpers instead.
