# 160PEROPTPREV-001: Consolidate `emptyPreviewUsage` and extend `SELECTION_REASONS`

**Status**: COMPLETED
**Priority**: HIGH
**Effort**: Small
**Engine Changes**: Yes — `agents/policy-eval.ts`, `agents/policy-agent.ts`
**Deps**: `specs/160-per-option-preview-inner-microturns.md`

## Problem

Spec 160 builds on a single source of truth for the `previewUsage` "disabled" shape, and the chooseN beam preview surfaces a new `selectionReason: 'beamPruned'` value. Today:

- `emptyPreviewUsage` is defined twice as module-private functions: `policy-agent.ts:144` (no-arg) and `policy-eval.ts:1226` (takes `AgentPreviewMode`). The duplication violates Foundation 15 (Architectural Completeness).
- `SELECTION_REASONS` at `policy-eval.ts:95` is a closed `as const` of six values; the new `'beamPruned'` value used by Spec 160's beam preview must extend it.

This ticket performs the refactor. No behavior change. It unblocks tickets 006 (beam preview emits `beamPruned`) and 007 (`chooseFrontierDecision` consumes the consolidated previewUsage).

## Assumption Reassessment (2026-05-06)

1. `emptyPreviewUsage` is duplicated at the cited locations (verified via reassess-spec; spec §6 records the duplication).
2. `SELECTION_REASONS` is closed at `policy-eval.ts:95` with six values; the constant uses `as const` and the type derives via `typeof SELECTION_REASONS[number]`.
3. The `policy-eval.ts:1226` definition takes `AgentPreviewMode` — a strictly more general signature than the no-arg variant in `policy-agent.ts:144`.

## Architecture Check

1. **Single source of truth** (Foundation 15): one canonical `emptyPreviewUsage(mode: AgentPreviewMode)` in `policy-eval.ts`. The `policy-agent.ts` callers pass `'disabled'` to obtain the shape they need today; future callers (ticket 007) pass other modes.
2. **No backwards-compatibility shim** (Foundation 14): the `policy-agent.ts` local definition is deleted, not aliased.
3. **Engine-agnostic** (Foundation 1): the `'beamPruned'` enum extension is generic — no game identifiers introduced.

## What to Change

### 1. Promote `emptyPreviewUsage` in `policy-eval.ts`

Add `export` to the existing `function emptyPreviewUsage(mode: AgentPreviewMode)` at `packages/engine/src/agents/policy-eval.ts:1226`. The signature and body remain unchanged; only the export is new.

### 2. Remove local `emptyPreviewUsage` from `policy-agent.ts`

Delete the no-arg definition at `packages/engine/src/agents/policy-agent.ts:144-164`. Add an import for the canonical helper from `./policy-eval`. Update the call at line 122 (inside `chooseStructuralFrontierDecision`) to pass `'disabled'`.

### 3. Extend `SELECTION_REASONS` with `'beamPruned'`

In `packages/engine/src/agents/policy-eval.ts:95`, append `'beamPruned'` to the closed `as const` tuple:

```ts
export const SELECTION_REASONS = ['coverage', 'prior', 'shallowDelta', 'widening', 'cache', 'gated', 'beamPruned'] as const;
```

The derived `SelectionReason` type picks up the new value automatically; no consumer change is required at this stage (ticket 006 emits `'beamPruned'` against the extended enum).
