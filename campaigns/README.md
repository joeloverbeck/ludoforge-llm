# Campaigns

`campaigns/` contains long-running optimization and evaluation programs that operate on repository-owned specs, engine code, and profiling harnesses. These directories are the working surface for agent-evolution loops and performance-reduction work, not the canonical location for rule-authoritative game behavior.

Rule-authoritative data still lives in game specs and agent YAML under `data/games/**`. Architectural requirements remain governed by [docs/FOUNDATIONS.md](/home/joeloverbeck/projects/ludoforge-llm/docs/FOUNDATIONS.md).

## What Lives Here

- Agent-evolution campaigns that improve one mutable policy profile against fixed controls.
- Performance campaigns that measure and reduce compile or simulation cost.
- Campaign-local operating instructions in each campaign's `program.md`.
- Exploratory notes in each campaign's `musings.md`.

## Policy-Profile Regressions

Campaign work can change whether an evolved or baseline policy profile still converges on the repository's tracked witness seeds. Those regressions are tracked in the engine test corpus, not in campaign-local scripts:

- Witness files live under `packages/engine/test/policy-profile-quality/`.
- The explicit local lane is `pnpm -F @ludoforge/engine test:policy-profile-quality`.
- CI runs the non-blocking `policy-profile-quality` job in [.github/workflows/engine-determinism.yml](/home/joeloverbeck/projects/ludoforge-llm/.github/workflows/engine-determinism.yml).
- The reporting surface is generated by [emit-policy-profile-quality-report.mjs](/home/joeloverbeck/projects/ludoforge-llm/packages/engine/scripts/emit-policy-profile-quality-report.mjs), which emits `POLICY_PROFILE_QUALITY_REGRESSION` warnings and a sticky PR summary.

That separation is intentional. Determinism failures are engine bugs; policy-profile-quality failures are profile-maintainer signals.

## Profile Lifecycle

An evolved profile is temporary campaign state. When an evolved profile becomes the new baseline, repository-owned references should be renamed in the same change rather than preserved through compatibility aliases. That includes the corresponding policy-profile-quality witness file when its variant identity changes.

Spec 136 defines the current separation between determinism proofs and policy-profile-quality witnesses:
