# 100COMEVEEFF-002: Add cardAnnotationIndex JSON Schema

**Status**: PENDING
**Priority**: HIGH
**Effort**: Small
**Engine Changes**: Yes — kernel schemas (`schemas-core.ts`)
**Deps**: `tickets/100COMEVEEFF-001.md`

## Problem

The `cardAnnotationIndex` field on `GameDef` needs a JSON Schema definition so that GameDef validation (used by tests, CLI, and the schema artifact pipeline) recognizes and validates the new field. Without the schema, compiled GameDefs with `cardAnnotationIndex` would fail validation.

## Assumption Reassessment (2026-03-31)

1. `schemas-core.ts` at `packages/engine/src/kernel/schemas-core.ts` contains the GameDef JSON Schema with existing optional fields like `cardMetadataIndex`. Confirmed.
2. Schema artifact generation uses `pnpm turbo schema:artifacts` to sync schemas. Confirmed via `packages/engine/scripts/`.
3. The `cardMetadataIndex` schema pattern provides a template for the new field.

## Architecture Check

1. Schema follows the same pattern as `cardMetadataIndex` — optional top-level field with a record of entries keyed by card ID.
2. No game-specific logic in the schema. All fields are generic numeric/boolean/string-array types.
3. No backwards-compatibility shims — the field is optional, so existing GameDef JSON files validate without it.

## What to Change

### 1. Add `CompiledEventSideAnnotation` schema

Define a JSON Schema object for `CompiledEventSideAnnotation` with all fields from the TypeScript interface:
- `tokenPlacements`, `tokenRemovals`, `tokenCreations`, `tokenDestructions`: objects with `additionalProperties: { type: 'number' }`
- Numeric fields: `markerModifications`, `globalMarkerModifications`, `globalVarModifications`, `perPlayerVarModifications`, `varTransfers`, `drawCount`, `shuffleCount`, `effectNodeCount`
- Boolean fields: `grantsOperation`, `hasEligibilityOverride`, `hasLastingEffect`, `hasBranches`, `hasPhaseControl`, `hasDecisionPoints`
- `grantOperationSeats`: array of strings

### 2. Add `CompiledEventCardAnnotation` schema

Object with:
- `cardId`: required string
- `unshaded`: optional `CompiledEventSideAnnotation`
- `shaded`: optional `CompiledEventSideAnnotation`

### 3. Add `CompiledEventAnnotationIndex` schema

Object with:
- `entries`: required object with `additionalProperties` referencing `CompiledEventCardAnnotation`

### 4. Add `cardAnnotationIndex` to GameDef schema

Add as an optional property on the GameDef schema object, referencing the `CompiledEventAnnotationIndex` schema.

### 5. Sync schema artifacts

Run `pnpm turbo schema:artifacts` to regenerate the JSON Schema artifact files.
