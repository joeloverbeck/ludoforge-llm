## Problem

147AOTCON-001 lands the `CompiledPolicyExpr` descriptor surface for the first supported expression families (the minimum subset defined in Spec 147 Â§D1) plus the equivalence-scaffold test (D5.1). It deliberately populates `AgentPolicyCatalog.compiled.considerations` only. The full descriptor union must cover every `AgentPolicyExpr.kind` and every AST-bearing library expression carrier currently emitted by `lowerAgents` before the compiled path can be enabled as default in 147AOTCON-003. Coverage gaps include the four aggregate kinds (`zoneTokenAgg`, `globalTokenAgg`, `globalZoneAgg`, `adjacentTokenAgg`), the structural variants (`zoneProp`, `seatAgg`), the 22 operators inside `case 'op'`, and non-consideration carriers (`stateFeatures`, `candidateFeatures`, `candidateAggregates`, `pruningRules`, `tieBreakers`, and `strategicConditions`).

Spec 147 also requires a determinism invariant test (D5.2) that compiles the FITL and Texas catalogs twice and asserts byte-identical descriptor trees and identical `catalogFingerprint` values. This test is the F#8/F#16 proof that the lowering pass is referentially transparent.

## Architecture Check

1. The descriptor union extension is mechanical: each new variant mirrors an existing `AgentPolicyExpr.kind`. No game-specific branches enter; the kernel materializes closures via the same fixed factory introduced in 001.
2. F#7 (Specs Are Data): every new descriptor variant and carrier map remains a JSON-serializable typed record. No `eval`, no `Function`, no closures stored on disk.
3. F#8 (Determinism): the determinism invariant test is the proof obligation for this ticket â€” once added, future regressions in lowering-pass purity will fail CI.
4. F#13 (Artifact Identity): the extended descriptors flow through the same `fingerprintPolicyIr` pipeline as the rest of the catalog. No new hash mechanism is introduced.
5. F#14 (No Backwards Compatibility): this ticket extends but does not enable the compiled path. The AST fallback remains in place because 003 atomically removes it alongside the fixture migration. This intermediate state is part of the migration sequence, not a permanent shim.

## What to Change

### 1. Extend the `CompiledPolicyExpr` union in `kernel/types-core.ts`

Add the missing variants to the `CompiledPolicyExpr` union introduced in 001 (per Spec 147 Â§D1):

- `{ kind: 'zoneProp'; zone: CompiledPolicyExpr | string; prop: string }`
- `{ kind: 'zoneTokenAgg'; zone: AgentPolicyZoneSource; owner: AgentPolicyZoneTokenAggOwner; prop: string; aggOp: AgentPolicyZoneTokenAggOp }`
- `{ kind: 'globalTokenAgg'; tokenFilter?: AgentPolicyTokenFilter; aggOp: AgentPolicyZoneTokenAggOp; prop?: string; zoneFilter?: AgentPolicyZoneFilter; zoneScope: AgentPolicyZoneScope }`

### 2. Extend the compiled catalog beyond considerations

Extend `CompiledPolicyCatalog` and its schema so the compiled side has a descriptor equivalent for every AST-bearing library carrier that 003 must later consume without `AgentPolicyExpr`:

- `stateFeatures[featureId].expr`
- `candidateFeatures[featureId].expr`
- `candidateAggregates[aggregateId].of` and optional `where`

- `considerations[considerationId].when` / `weight` / `value`
- `tieBreakers[tieBreakerId].value`
- `strategicConditions[conditionId].target` and optional `proximity.current`

Preserve each carrier's existing metadata (`type`, `costClass`, `dependencies`, aggregate `op`, consideration clamp/unknown behavior, etc.) so 003 can remove AST-bearing fields without inventing a second migration shape.

### 3. Extend `lowerAgentConsiderations` / compiled-policy lowering

### 4. Extend `buildPolicyExprClosure` in `agents/compiled-policy-runtime.ts`
