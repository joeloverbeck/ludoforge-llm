# 147AOTCON-003: Enable compiled policy path as default, delete AST interpreter, regenerate fixtures, re-measure hard-target

**Status**: PENDING
**Priority**: HIGH
**Effort**: Large
**Engine Changes**: Yes ‚Äî agent runtime, kernel types, schema, four golden fixtures, hard-target measurement
**Deps**: `archive/tickets/147AOTCON-002.md`

## Problem

After 147AOTCON-001 and 002, the `compiled` descriptor tree exists alongside the AST-bearing `library` shape and covers every `AgentPolicyExpr.kind`. The runtime still dispatches via the AST interpreter (`evaluateExpr` in `policy-evaluation-core.ts:446`) when the compiled tree is absent ‚Äî a transitional state. Spec 147 ¬ßD4 and the Foundation 14 alignment in ¬ßD6 require the AST interpreter to be deleted, the optional marker on `AgentPolicyCatalog.compiled` to be removed, and the four repository-owned golden fixtures to be regenerated **in the same change**, not as a sequence of compatibility steps.

This ticket also satisfies Acceptance Criteria 1 (‚â•6% drop in `previewOn_totalMs_ms`) and 5 (‚â•50% drop in `resolveRef` self-time on a `--cpu-prof` measurement) by re-running `pnpm -F @ludoforge/engine measure:preview-hard-target` and recording pre/post profiles.

## Assumption Reassessment (2026-04-26)

1. `policy-evaluation-core.ts:407-444` (`evaluateConsideration`) currently calls `evaluateExpr` three times per consideration (`when`, `weight`, `value`). After 002 lands, every consideration has a `CompiledPolicyConsideration` parallel ‚Äî making `evaluateExpr` redundant once the compiled-only path is enabled.
2. `policy-evaluation-core.ts:781-823` (`resolveRef`) is only called from `evaluateExpr`'s `case 'ref'` arm (the agent confirmed no other callers in the reassessment). Deleting `evaluateExpr` orphans `resolveRef` ‚Äî it must be deleted in the same change.
3. The four golden fixtures (`fitl-policy-catalog.golden.json`, `texas-policy-catalog.golden.json`, `fitl-policy-summary.golden.json`, `texas-policy-summary.golden.json`) currently encode the AST-bearing `library` shape. Per F#14, they must be regenerated to encode the compiled-only shape in the same change as the type narrowing.
4. The hard-target measurement command (`scripts/measure-preview-pipeline-hard-target.mjs`, added under archived ticket 146DRIVE-005) is the canonical methodology for Acceptance Criteria 1 and 5 ‚Äî same script, same corpus, before/after profile comparison.
5. The library's `expr: AgentPolicyExpr` fields (on stateFeatures, candidateFeatures, aggregates, strategicConditions) are no longer read at runtime once the compiled path is the only path. The spec's Follow-On Tickets entry calls out "the AST-bearing `library` shape that's no longer needed" ‚Äî this ticket removes those fields from the runtime types.

## Architecture Check

1. **Foundation 14 atomic cut**: The deletion of `evaluateExpr` + `resolveRef` + AST-bearing library fields + the four fixture regenerations is a single coherent change. Splitting it would leave the build in a state where source and fixtures disagree (compiled-only types vs. AST-shaped fixture data). The Large effort rating reflects the deletion blast radius across the 4 fixtures + ~40 fixture-construction sites in tests; the change is mechanically uniform (each fixture regenerates via the existing `sync-fixtures.sh` workflow), satisfying the Foundation 14 mechanical-uniformity exception in `tickets/_TEMPLATE.md` Effort calibration.
2. **No transitional shim**: After this ticket lands, no `?` optional marker remains on `AgentPolicyCatalog.compiled`. `GameDef.agents`, when present, MUST carry a `compiled` field. Type-system-enforced ‚Äî readers cannot accidentally fall through to a missing-compiled branch.
3. **Engine agnosticism**: The deletion path touches only kernel/agent types and game-agnostic compiled descriptors. No game-specific knowledge enters the engine.
4. **Determinism preserved**: The compiled path's referential-transparency proof (147AOTCON-002 D5.2) gates this ticket ‚Äî if 002's determinism test fails, 003 must not land. Replay identity across 200-turn FITL traces is the runtime determinism gate.

## What to Change

### 1. Make `compiled` required in `AgentPolicyCatalog`

In `packages/engine/src/kernel/types-core.ts`, change `AgentPolicyCatalog.compiled` from optional to required. Remove the `expr: AgentPolicyExpr` fields from `CompiledAgentStateFeature`, `CompiledAgentCandidateFeature`, `CompiledAgentAggregate`, and `CompiledStrategicCondition` (the library shapes that previously carried AST). The compiled descriptor tree is the only remaining expression carrier.

### 2. Delete the AST interpreter

In `packages/engine/src/agents/policy-evaluation-core.ts`:

- Delete `evaluateExpr` (lines 446‚Äì562).
- Delete `resolveRef` (lines 781‚Äì823) ‚Äî orphaned by `evaluateExpr` deletion.
- Delete `evaluateExprList` and any other helpers exclusive to the AST path.
- Reduce `evaluateConsideration` (lines 407‚Äì444) to the compiled-only path: read `CompiledPolicyConsideration` from `this.input.def.agents.compiled.considerations[considerationId]`, materialize closures via `buildPolicyExprClosure` (147AOTCON-001's runtime), and evaluate. The dispatcher branch from D4 collapses to a single path.

Update the four direct callers of `evaluateExpr` outside `policy-evaluation-core.ts` (per the reassessment Agent 1 inventory):

- `agents/policy-eval.ts` (2 tieBreaker call sites) ‚Äî switch to `buildPolicyExprClosure`.
- `test/integration/agents/strategic-condition-e2e.test.ts` (11 test call sites) ‚Äî switch to compiled-path equivalents.
- `test/unit/agents/policy-eval-strategic-condition.test.ts` (1 test call site) ‚Äî same.

### 3. Update `lowerAgents` to drop AST emission

In `packages/engine/src/cnl/compile-agents.ts`:

- Stop emitting `expr: AgentPolicyExpr` fields in the library output (since the types no longer carry them).
- The `lowerAgentConsiderations` pass (introduced 001, extended 002) becomes the sole expression-emission path.
- `catalogFingerprint` still computed from the assembled catalog (now compiled-only) by `fingerprintPolicyIr`.

Run `pnpm -F @ludoforge/engine schema:artifacts`. Confirm `packages/engine/schemas/GameDef.schema.json` no longer contains AST-shaped `expr` fields under `library.*`, and that `compiled` is required under `agents`.
