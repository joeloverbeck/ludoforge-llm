# 147AOTCON-004: Remove Zobrist and GC bottlenecks from FITL preview hard-target path

**Status**: PENDING
**Priority**: HIGH
**Effort**: Large
**Engine Changes**: Yes — preview pipeline performance, Zobrist/state identity hot path, allocation pressure
**Deps**: `archive/tickets/147AOTCON-003.md`

## Problem

147AOTCON-003 removed the agent policy AST interpreter from the runtime path and made `AgentPolicyCatalog.compiled` required, but the hard-target probe still misses the FITL preview target:

```text
previewOn_totalMs_ms=28946.57,27447.55,27258.11
mean_totalMs=27884.08
mad_pct=2.54
hardTargetMs=25600
pass=false
candidateBudget=465
sampledActionSelectionCount=50
```

The post-migration CPU profile under `/tmp/147aotcon-profile-post/CPU.20260426.173936.2.0.001.cpuprofile` shows the remaining top self-time samples dominated by garbage collection and Zobrist hashing (`fnv1a64`, `digestDecisionStackFrame`, `zobristKey`), not the deleted policy AST interpreter. Per Foundations 15 and 16, the hard-target miss must be handled as a root-cause performance ticket rather than hidden under the AOT policy migration.

## Assumption Reassessment (2026-04-26)

1. The AOT policy seam is no longer the dominant blocker after 147AOTCON-003; the live profile points at state identity/hash churn and allocation pressure.
2. The benchmark corpus remains the existing hard-target script: `packages/engine/scripts/measure-preview-pipeline-hard-target.mjs --runs 3`, with `candidateBudget=465` and `sampledActionSelectionCount=50`.
3. Direct evaluator micro-optimizations remain risky. Spec 147 records prior evidence that kernel hot-path tweaks can deopt; this ticket must first prove which work can be skipped or amortized at the orchestration/state-identity layer.

## Architecture Check

1. **Foundation 8 determinism**: Hashes may accelerate comparison, but canonical serialized state remains the equality oracle. Any hash optimization must preserve byte-identical replay state for the same GameDef, initial state, seed, and actions.
2. **Foundation 11 immutability**: Any allocation reduction must respect the external immutable-state contract. Scoped internal mutation is allowed only if isolated and regression-tested.
3. **Foundation 15 root cause**: The work targets the measured bottleneck (`fnv1a64`/Zobrist/GC), not the already-removed policy interpreter.
4. **Foundation 16 proof**: The hard target is not considered satisfied until the same 3-run script passes and replay/determinism tests remain green.

## What to Change

### 1. Attribute Zobrist and GC cost precisely

Inspect the post-147AOTCON-003 CPU profile and add a focused diagnostic note or test helper that identifies whether the repeated hashing is caused by decision stack frames, whole-state hash recomputation, preview candidate cloning, or canonical serialization churn.

### 2. Remove work at the orchestration/state-identity layer

Implement the smallest architecture-aligned optimization that skips or amortizes repeated Zobrist/hash work in the preview hard-target path without changing rule semantics. Candidate directions include caching immutable frame digests, avoiding duplicate candidate identity recomputation, or reusing already-derived state identity within one preview evaluation scope.

### 3. Prove determinism and hard-target recovery

Re-run the same hard-target profile command and record the mean, MAD, candidate budget, sample count, and top self-time samples. Replay/determinism proof must remain byte-identical.
